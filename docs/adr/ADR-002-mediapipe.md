# ADR-002: AI 顔検出ライブラリとして MediaPipe FaceDetection を採用する

## Context

SRS FR-002 は顔写真から目・鼻・口・眉・耳を個別パーツとして自動切り出しする機能を要求している。
SRS NFR-006/007 は顔写真データを外部に送信することを禁止し、セッション終了後のデータ削除を要求している。
SRS C-002/003 は無償かつ既存ライブラリの利用を制約としている。
SRS TBD-001 においてライブラリ選定が本フェーズの課題として挙げられていた。

## Options

| 選択肢 | ブラウザ内完結 | 無償 | ランドマーク検出精度 | 備考 |
|---|---|---|---|---|
| MediaPipe FaceDetection | はい | はい | 顔枠 + 6点ランドマーク(目・鼻・口・耳) | Google 公式。CDN 経由で利用可能 |
| face-api.js | はい | はい | 68点ランドマーク(高精度) | 開発が低活発。バンドルサイズが大きい |
| AWS Rekognition API | いいえ(外部送信) | 無償枠あり | 高精度 | NFR-007 違反。サーバーレス構成と不適合 |

## Decision

**MediaPipe FaceDetection (CDN 経由) を採用する。**

## Rationale

- ブラウザ内でモデル推論が完結するため、NFR-006/007 を構造的に満たす
- Google が公式にメンテナンスしており、長期的なサポートが期待できる
- CDN 経由で利用できるため、ビルド環境が不要なプロジェクト構成と整合する
- 6 点ランドマーク(両目、鼻先、口中央、両耳)から各パーツの矩形領域を算出し、Canvas で切り出す実装が可能
- face-api.js と比較してバンドルサイズが小さく、モバイル端末での初回ロードが速い

## Consequences

- 肯定的影響:顔写真データが端末外に出ないことが技術的に保証される
- 否定的影響:ランドマークが 6 点のみであるため、「眉」の検出精度が低い場合がある。眉が検出不可の場合はスコープ縮小(TBD-003)で対応する
- 否定的影響:CDN 障害時にライブラリが取得不可になるリスクがある(R-002)。本番リリース時にライブラリをリポジトリに同梱する対策を検討する
- 中立:初回ロード時にモデルファイル(約 4MB)をダウンロードするため、Wi-Fi 環境前提の P-002 を前提条件として維持する
